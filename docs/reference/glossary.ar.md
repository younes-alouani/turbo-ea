# مسرد المصطلحات

| المصطلح | التعريف |
|------|------------|
| **سجل قرار الهندسة (ADR — Architecture Decision Record)** | مستند رسمي يلتقط قرار هندسة مهمًّا، بما في ذلك السياق ومبرّر القرار والعواقب والبدائل المدروسة. تدعم سجلات ADR سير عمل توقيع وسلسلة مراجعات |
| **حالة الموافقة** | حالة مراجعة البطاقة: مسودة، مُعتمد، معطوب، أو مرفوض. تتحوّل البطاقات المُعتمدة إلى معطوبة عند تحريرها |
| **إشارة مرجعية / عرض محفوظ** | تهيئة محفوظة للمرشّحات والأعمدة والفرز في المخزون يمكن إعادة تحميلها بنقرة واحدة |
| **BPM** | إدارة عمليات الأعمال — انضباط نمذجة عمليات الأعمال وتحليلها وتحسينها |
| **ترتيب صفوف BPM** | ترتيب عرض صفوف أنواع العمليات (Core، Support، Management) في متصفّح عمليات BPM، قابل للتهيئة بسحب الصفوف |
| **BPMN** | نموذج وترميز عمليات الأعمال — الترميز القياسي لنمذجة عمليات الأعمال (الإصدار 2.0) |
| **قدرة الأعمال** | ما يمكن للمنظمة فعله، بغضّ النظر عن كيفية فعله |
| **حساب** | صيغة معرّفة من قبل المسؤول تحسب قيمة حقل تلقائيًا عند حفظ البطاقة |
| **بطاقة** | الوحدة الأساسية للمعلومات في Turbo EA التي تمثّل أي مكوّن هندسة |
| **نوع البطاقة** | الفئة التي تنتمي إليها البطاقة (مثل تطبيق، عملية أعمال، منظمة) |
| **درجة الثقة** | تقييم من 0–100% يشير إلى مدى موثوقية وصف مُولَّد بالذكاء الاصطناعي |
| **جودة البيانات** | درجة اكتمال من 0–100% تستند إلى الحقول المعبّأة وأوزانها المهيّأة |
| **مخطط** | مخطط هندسة مرئي يُنشأ بمحرّر DrawIO المضمَّن |
| **مرفق ملف** | ملف ثنائي (PDF، DOCX، XLSX، صور، حتى 10 ميغابايت) يُرفع مباشرة إلى بطاقة عبر تبويب الموارد |
| **السنة المالية** | فترة الـ 12 شهرًا المستخدمة للموازنة والتقارير المالية. قابلة للتهيئة عبر الإدارة > الإعدادات — يحدّد شهر البداية (من يناير حتى ديسمبر) كيفية تجميع بنود ميزانية PPM |
| **DrawIO** | أداة الرسم البياني مفتوحة المصدر المضمَّنة المستخدمة لمخططات الهندسة المرئية |
| **بند تكلفة** | إدخال تكلفة ميزانية أو فعلية (CapEx/OpEx) في مبادرة PPM، يُستخدم لتتبّع الإنفاق المالي |
| **هندسة المؤسسة (EA — Enterprise Architecture)** | الانضباط الذي ينظّم ويوثّق بنية أعمال المنظمة وتقنيتها |
| **EOL (End of Life — نهاية العمر)** | التاريخ الذي يفقد فيه منتج تقني دعم المزوّد. يُتتبَّع عبر التكامل مع endoflife.date |
| **مخطط Gantt** | خط زمني مرئي بأشرطة أفقية تُظهر جدول المشروع ومدته وتقدّمه لكل حزمة عمل ومهمة |
| **مبادرة** | مشروع أو برنامج ينطوي على تغييرات في الهندسة |
| **دورة الحياة** | المراحل الخمس التي يمرّ بها المكوّن: Plan، Phase In، Active، Phase Out، End of Life |
| **LLM** | نموذج لغوي كبير — نموذج ذكاء اصطناعي يولّد نصًا (مثل Ollama، OpenAI، Anthropic Claude، Google Gemini) |
| **MCP** | بروتوكول سياق النموذج — معيار مفتوح يتيح لأدوات الذكاء الاصطناعي (Claude، Copilot، Cursor) الاتصال بمصادر بيانات خارجية. يوفّر خادم MCP المدمج في Turbo EA وصولًا للقراءة فقط إلى بيانات هندسة المؤسسة مع RBAC لكل مستخدم |
| **النموذج الفوقي** | النموذج القائم على البيانات الذي يعرّف بنية المنصة: أنواع البطاقات والحقول والعلاقات والأدوار |
| **معلَم بارز** | حدث مهم أو نقطة إنجاز في خط زمني للمشروع، يُعرض كمؤشّر ماسي في مخطط Gantt |
| **إشعار** | تنبيه داخل التطبيق أو عبر البريد الإلكتروني يُطلَق بأحداث النظام (تعيين مهمة، تحديث بطاقة، إضافة تعليق، إلخ) |
| **تغذية OData** | تغذية بيانات JSON متاحة على عروض المخزون المحفوظة (الإشارات المرجعية) للاستهلاك بأدوات خارجية مثل Power BI أو Excel |
| **Ollama** | أداة مفتوحة المصدر لتشغيل نماذج LLM محليًا على أجهزتك الخاصة |
| **محفظة** | مجموعة من التطبيقات أو التقنيات تُدار كمجموعة |
| **PPM** | إدارة محفظة المشاريع — انضباط إدارة محفظة من المشاريع والمبادرات بالميزانيات والمخاطر والمهام وتقارير الحالة |
| **رقم مرجعي** | معرّف تسلسلي مُولَّد تلقائيًا لسجلات ADR (مثل ADR-001، ADR-002) يوفّر تسمية فريدة قابلة للقراءة البشرية |
| **مهمة متكرّرة** | مهمة بطاقة مضبوطة على التكرار وفق جدول (كل N أيام/أسابيع/أشهر/سنوات). إكمالها يُنشئ التكرار التالي؛ تبقى التكرارات البعيدة المستقبل "مجدولة" حتى تُفتح نافذة المهلة، عندئذ تظهر في صفحة المهام وتُخطر المُعيَّن إليه |
| **علاقة** | اتصال بين بطاقتين يصف كيف ترتبطان (مثل "يستخدم"، "يعتمد على"، "يعمل على") |
| **تبويب الموارد** | تبويب في تفاصيل البطاقة يجمع سجلات قرارات الهندسة ومرفقات الملفات وروابط المستندات في مكان واحد |
| **حالة RAG** | مؤشّر صحة أحمر-كهرماني-أخضر يُستخدم في تقارير حالة PPM للجدول والتكلفة والنطاق (مُعيَّن إلى خارج المسار، في خطر، على المسار) |
| **درجة المخاطرة** | قيمة محسوبة تلقائيًا (الاحتمال × الأثر) تقيس شدة مخاطرة مشروع في سجل مخاطر PPM |
| **تقرير محفوظ** | تهيئة تقرير ثابتة بمرشّحات ومحاور وإعدادات تصوّر يمكن إعادة تحميلها |
| **مراجعة (ADR)** | نسخة جديدة من ADR موقَّع ترث المحتوى وروابط البطاقات من النسخة السابقة، مع رقم مراجعة متزايد |
| **قسم** | منطقة قابلة للتجميع في صفحة تفاصيل البطاقة تحتوي على حقول مرتبطة، قابلة للتهيئة لكل نوع بطاقة |
| **موقِّع** | مستخدم مُعيَّن لمراجعة مستند ADR أو SoAW والتوقيع عليه. يتتبّع سير عمل التوقيع التواقيع المعلّقة والمكتملة |
| **SoAW** | بيان عمل الهندسة — مستند TOGAF رسمي يحدّد النطاق والمُخرَجات لمبادرة |
| **SSO** | الدخول الموحّد — تسجيل الدخول باستخدام بيانات اعتماد المؤسسة عبر مزوّد هوية (Microsoft، Google، Okta، OIDC) |
| **نوع فرعي** | تصنيف ثانوي ضمن نوع بطاقة (مثل: Application له أنواع فرعية: Business Application، Microservice، AI Agent، Deployment). يعمل كل نوع فرعي كقالب فرعي يمكنه التحكّم في رؤية الحقول |
| **قالب النوع الفرعي** | تهيئة أي الحقول مرئية أو مخفية لنوع فرعي محدّد. يهيّئ المسؤولون هذا في إدارة النموذج الفوقي بالنقر على رقاقة نوع فرعي |
| **صاحب مصلحة** | شخص له دور محدّد على بطاقة (مثل مالك التطبيق، المالك التقني) |
| **استبيان** | استبيان صيانة بيانات يستهدف أنواع بطاقات محدّدة لجمع المعلومات من أصحاب المصلحة |
| **وسم / مجموعة وسوم** | تسمية تصنيف منظَّمة في مجموعات بأوضاع تحديد فردي أو متعدد، مع قيود نوع اختيارية، وعلامة إلزامية اختيارية تحجب الموافقة وتغذّي درجة جودة البيانات |
| **مجموعة وسوم إلزامية** | مجموعة وسوم مُعلَّمة كمطلوبة. لا يمكن اعتماد البطاقات المنطبقة حتى يُرفق وسم واحد على الأقل من المجموعة، وتلبية ذلك تساهم في درجة جودة بيانات البطاقة |
| **الكشف الدلالي لقانون EU AI Act** | مرور امتثال في TurboLens يطلب من نموذج LLM تعليم البطاقات التي تتضمّن قدرات ذكاء اصطناعي / تعلّم آلي (نماذج LLM، محرّكات التوصية، رؤية الحاسوب، التسجيل، روبوتات المحادثة، …) حتى عندما لا تكون مصنّفة صراحةً كـ `AI Agent` / `AI Model`. تُعلَّم هذه النتائج **مكتشَفة بالذكاء الاصطناعي** |
| **المخاطرة الأولية مقابل المتبقية** | تقييمان يُلتقطان على كل مخاطرة في سجل المخاطر. `Initial` هي الاحتمال × الأثر غير المُخفَّف؛ `Residual` هي الاحتمال × الأثر بعد التخفيف، قابلة للتحرير بمجرد وجود خطة تخفيف. يشتقّ كلاهما مستوى عبر مصفوفة 4×4 |
| **مرجع المخاطرة** | معرّف تصاعدي قابل للقراءة البشرية (`R-000123`) يُعيَّن عند إنشاء المخاطرة. يبقى مرئيًا في أزرار النتيجة المُرقّاة (**Open risk R-000123**) وفي وصف المهمة المرتبطة بالمالك |
| **TOGAF** | إطار عمل هندسة The Open Group — منهجية هندسة مؤسسة واسعة الاستخدام. تتوافق ميزة SoAW في Turbo EA مع TOGAF |
| **تقرير الحالة** | تقرير PPM شهري يتتبّع صحة المشروع عبر مؤشّرات RAG للجدول والتكلفة والنطاق، إلى جانب الإنجازات والخطوات التالية |
| **بوابة إلكترونية** | عرض عام للقراءة فقط لبطاقات مختارة يمكن الوصول إليه دون مصادقة عبر URL فريد |
| **هيكل تجزئة العمل (WBS — Work Breakdown Structure)** | تفكيك هرمي لنطاق المشروع إلى حزم عمل، لكلٍّ تواريخ بدء/انتهاء وتتبّع إنجاز. يُستخدم في مخطط Gantt لـ PPM |
| **حزمة عمل** | تجميع منطقي للمهام ضمن خط زمني في Gantt له تاريخ بدء وتاريخ انتهاء ونسبة إنجاز خاصة به |
| **اقتراح الذكاء الاصطناعي** | وصف بطاقة مُولَّد تلقائيًا يُنتَج بدمج نتائج بحث الويب مع نموذج لغوي كبير (LLM) |
| **حكم الذكاء الاصطناعي** | تأكيد المستخدم أو رفضه لتصنيف نموذج LLM لبطاقة على أنها حاملة للذكاء الاصطناعي (`hasAiFeatures = true / false`). يبقى عبر عمليات إعادة الفحص بحيث لا يستطيع انحراف LLM تغيير نطاق EU AI Act بصمت |
| **GRC** | الحوكمة والمخاطر والامتثال — مساحة العمل الموحّدة في `/grc` بثلاثة تبويبات (الحوكمة، المخاطر، الامتثال) تجمع مبادئ هندسة المؤسسة وسجلات ADR وسجل المخاطر وماسح الأمان والامتثال |
| **المرحلة G** | مرحلة "حوكمة التنفيذ" في TOGAF ADM. مصدر مفردات سجل المخاطر ودورة حياته |
| **سجل المخاطر** | سجل على مستوى المشهد لمخاطر الهندسة متوائم مع TOGAF المرحلة G. يقع في `/grc?tab=risk`. متميّز عن المخاطر المحصورة بالمبادرة المُلتقطة داخل PPM |
| **مالك المخاطرة** | المستخدم المسؤول عن مخاطرة. يُنشئ التعيين تلقائيًا مهمة نظام في صفحة مهام المالك ويُطلق إشعار `risk_assigned` |
| **مهمة التخفيف** | عنصر عمل مملوك مرفق بمخاطرة يلتقط نشاط تخفيف ملموسًا. يمكن أن تكون لمرة واحدة أو متكرّرة (يومية / أسبوعية / شهرية / سنوية). تتدحرج المهام المتكرّرة إلى الأمام بشكل صحيح تقويميًا عند الإكمال |
| **تكرار مهمة التخفيف** | نسخة مجدولة واحدة من مهمة تخفيف. تتحرّك عبر `scheduled` ← `open` ← `done` / `skipped`. تلتقط لقطة للمُعيَّن إليه عند الفتح والمالك-عند-الإكمال عند الإغلاق بحيث تنجو إجابات التدقيق من تناوب الملّاك |
| **المهلة (مهمة التخفيف)** | عدد الأيام قبل `due_date` التي تُرقّى فيها دورة مجدولة إلى `open` وتهبط في مهام المُعيَّن إليه. قيم افتراضية ذكية لكل وحدة (1 / 2 / 7 / 14 لليومية / الأسبوعية / الشهرية / السنوية) مُقيَّدة بنصف الدورة |
| **نتيجة امتثال** | صف في سجل الامتثال مقابل لائحة × مادة. يكتبه مراجع يدويًا أو يُنتجه فحص TurboLens AI؛ يتشارك النوعان دورة الحياة نفسها ويمكن ترقيتهما إلى مخاطرة |
| **قدرة كلّية** | تجميع المستوى-0 فوق L1 في كتالوج القدرات. تهبط كبطاقة `BusinessCapability` مع `attributes.capabilityLevel = "Macro"` و`catalogueId` ببادئة `MC-`. تُرخّي حد عمق التسلسل الهرمي إلى 6 |
| **عرض التبعية الطبقي (LDV — Layered Dependency View)** | الترميز الخاص بـ Turbo EA لمخططات التبعية: بطاقات مجمَّعة في طبقات هندسة المؤسسة الأربع كمسارات سباحة، ملوَّنة بنوع البطاقة، مع عرض البطاقات المقترَحة كعُقد ذات حدود متقطّعة وشارة "NEW" خضراء. يُستخدم في تقرير التبعيات، وقسم تبعية تفاصيل البطاقة، والهندسة المستهدفة في TurboLens Architect |
| **TIME (التحمّل / الاستثمار / الترحيل / الإلغاء)** | إطار تصنيف محفظة رباعي للتطبيقات، شاعه Gartner. يُوسَم كل تطبيق بتصنيف واحد — Tolerate التحمّل (الإبقاء كما هو)، Invest الاستثمار (تمويل التحسينات)، Migrate الترحيل (الاستبدال أو إعادة الاستضافة)، أو Eliminate الإلغاء (إيقاف التشغيل). في Turbo EA يُضاف عادةً كحقل `single_select` على نوع بطاقة Application ويُستخدم كمحور اللون في تقرير المحفظة |
| **ترشيد محفظة التطبيقات** | أكثر مبادرات هندسة المؤسسة الأولى شيوعًا على Turbo EA: جرد التطبيقات في النطاق، وتصنيف كلٍّ بحسب قيمة الأعمال والملاءمة التقنية، وإسناد تصنيف TIME يقود قرارات الدمج / الاستبدال / الإيقاف |
| **الزحف-المشي-الجري** | نمط الطرح المرحلي الموصى به في دليل المبتدئين لهندسة المؤسسة. الزحف = نطاق ضيّق، التطبيقات فقط، خمسة حقول لكل بطاقة. المشي = إضافة ربط القدرات وأول تحليل محفظة. الجري = التوسّع إلى العمليات والواجهات والبيانات والوحدات المتقدمة |
