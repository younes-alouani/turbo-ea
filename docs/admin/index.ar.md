# الإدارة

![قائمة المستخدم مع خيارات الإدارة](../assets/img/en/19_user_menu.png)

يتوفّر قسم **الإدارة** للمستخدمين الذين يحملون دور **Admin**. وهو يوفّر أدوات لتهيئة نموذج بيانات المنصة، ووصول المستخدمين، والتكاملات، والإعدادات الشاملة للنظام.

| الصفحة | الوصف |
|------|-------------|
| [الإعدادات العامة](settings.md) | المظهر (الشعار، أيقونة المتصفح، العملة)، اللغات المُفعّلة، السنة المالية، البريد الإلكتروني (SMTP)، ومفاتيح تبديل الوحدات (BPM، PPM، GRC) |
| [النموذج الفوقي](metamodel.md) | أنواع البطاقات، والحقول، والأقسام، والأنواع الفرعية، وأنواع العلاقات، والحسابات، والوسوم، ومبادئ هندسة المؤسسة، ورسم النموذج الفوقي |
| [الحسابات](calculations.md) | الصيغ التي يحدّدها المسؤول والتي تحسب قيم الحقول تلقائيًا عند حفظ البطاقات |
| [الوسوم](tags.md) | مجموعات الوسوم والوسوم لتصنيف البطاقات بمرونة |
| [المستخدمون والأدوار](users.md) | إدارة المستخدمين، والأدوار على مستوى التطبيق، وتهيئة الأذونات |
| [المصادقة والدخول الموحّد](sso.md) | تهيئة الدخول الموحّد (Microsoft Entra ID، Google، Okta، Generic OIDC) وسير عمل الدعوات |
| [قدرات الذكاء الاصطناعي](ai.md) | اقتراحات وصف الذكاء الاصطناعي المدعومة بنماذج LLM (Ollama، المزوّدون المتوافقون مع OpenAI) مع سياق البحث على الويب |
| [تكامل MCP](mcp.md) | خادم Model Context Protocol لوصول أدوات الذكاء الاصطناعي (Claude Desktop، Copilot، Cursor) |
| [مرجع API](api.md) | مواصفة OpenAPI 3 الحيّة المضمّنة في دليل المستخدم (لا حاجة إلى نسخة خلفية)، إضافة إلى المصادقة والأذونات ونصائح توليد الشيفرة |
| [نهاية العمر (EOL)](eol.md) | ربط بيانات EOL بالجملة من endoflife.date إلى البطاقات التقنية |
| [الاستبيانات](surveys.md) | استبيانات صيانة البيانات الموجّهة إلى أنواع بطاقات وأصحاب مصلحة محددين |
| [البوابات الإلكترونية](web-portals.md) | بوابات عامة للقراءة فقط يمكن الوصول إليها دون مصادقة |
| [تكامل ServiceNow](servicenow.md) | مزامنة ثنائية الاتجاه بين بطاقات Turbo EA وقاعدة بيانات ServiceNow CMDB |
