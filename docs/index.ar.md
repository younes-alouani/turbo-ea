# مقدّمة إلى Turbo EA

### ما هو Turbo EA؟

**Turbo EA** هو منصة حديثة ذاتية الاستضافة لـ **إدارة هندسة المؤسسة**. وهي تمكّن المنظمات من توثيق جميع مكوّنات هندسة أعمالها وتقنيتها وتصوّرها وإدارتها في مكان واحد.

### لمن هذا الدليل؟

هذا الدليل مخصّص لـ **كل من يستخدم Turbo EA** — مهندسي المؤسسة، ومديري تقنية المعلومات، ومحلّلي الأعمال، والمطوّرين، والمسؤولين. سواء كنت تقيّم المنصة، أو تدير مشهد تقنية المعلومات في منظمتك يوميًا، أو تهيّئ النظام كمسؤول، ستجد المعلومات التي تحتاجها هنا. لا تُشترَط معرفة تقنية متقدّمة للبدء.

### الفوائد الرئيسية

- **رؤية شاملة**: عرض جميع التطبيقات والعمليات والقدرات والتقنيات عبر المنظمة في منصة واحدة.
- **اتخاذ قرارات مستنيرة**: تقارير مرئية (المحفظة، وخرائط القدرات، والتبعيات، ودورة الحياة، والتكلفة، والمزيد) تسهّل تقييم الحالة الراهنة للبنية التحتية التقنية.
- **إدارة دورة الحياة**: تتبّع حالة كل مكوّن تقني عبر خمس مراحل — من التخطيط حتى التقاعد.
- **التعاون**: يمكن لعدّة مستخدمين العمل في آنٍ واحد، مع أدوار قابلة للتهيئة، وإسناد أصحاب المصلحة، والتعليقات، والمهام، والإشعارات.
- **الأوصاف المدعومة بالذكاء الاصطناعي**: أنشئ أوصاف البطاقات بنقرة واحدة. يجمع Turbo EA بين البحث على الويب ونموذج LLM محلّي أو تجاري لإنتاج ملخّصات مدركة للنوع — مكتملة بدرجات الثقة وروابط المصادر. يعمل بالكامل على بنيتك التحتية حفاظًا على الخصوصية، أو اتّصل بمزوّدين تجاريين (OpenAI، Google Gemini، Anthropic Claude، والمزيد). خاضع للتحكّم الكامل من قِبل المسؤول: اختر أنواع البطاقات التي تحصل على اقتراحات الذكاء الاصطناعي، واختر مزوّد البحث، وحدّد النموذج.
- **المخططات المرئية**: أنشئ مخططات هندسية باستخدام محرّر DrawIO المضمّن، متزامنًا بالكامل مع مخزون بطاقاتك.
- **نمذجة عمليات الأعمال**: محرّر تدفّق عمليات بصيغة BPMN 2.0 مع ربط العناصر، وسير عمل الموافقة، وتقييمات النضج.
- **تكامل ServiceNow**: مزامنة ثنائية الاتجاه مع ServiceNow CMDB لإبقاء مشهد هندسة مؤسستك متّصلًا ببيانات عمليات تقنية المعلومات.
- **متعدّد اللغات**: متاح بالإنجليزية والإسبانية والفرنسية والألمانية والإيطالية والبرتغالية والصينية.

### المفاهيم الرئيسية

| المصطلح | المعنى |
|------|---------|
| **بطاقة** | العنصر الأساسي للمنصة. يمثّل أي مكوّن هندسي: تطبيقًا، عملية، قدرة أعمال، إلخ. |
| **نوع البطاقة** | الفئة التي تنتمي إليها البطاقة (تطبيق، عملية أعمال، منظمة، إلخ) |
| **العلاقة** | اتصال بين بطاقتين يصف كيفية ارتباطهما (مثل "uses" أو "depends on" أو "is part of") |
| **النموذج الفوقي** | البنية التي تحدّد أنواع البطاقات الموجودة، والحقول التي تمتلكها، وكيفية ارتباطها ببعضها. قابلة للتهيئة بالكامل من قِبل المسؤول |
| **دورة الحياة** | المراحل الزمنية لمكوّن: Plan، Phase In، Active، Phase Out، End of Life |
| **المخزون** | قائمة قابلة للبحث والتصفية بجميع البطاقات عبر كل نوع. التحرير بالجملة، والاستيراد والتصدير عبر Excel/CSV، والعروض المحفوظة مع المشاركة |
| **التقارير** | تصورات معدّة مسبقًا: المحفظة، وخريطة القدرات، ودورة الحياة، والتبعيات، والتكلفة، والمصفوفة، وجودة البيانات، ونهاية العمر الافتراضي |
| **BPM** | إدارة عمليات الأعمال — نمذجة عمليات الأعمال بمحرّر BPMN 2.0، وربط عناصر المخطط بالبطاقات، وتقييم النضج والمخاطرة والأتمتة |
| **PPM** | إدارة محفظة المشاريع — إدارة بطاقات المبادرات كمشاريع كاملة مع تقارير الحالة، وهياكل تجزئة العمل، ولوحات المهام بأسلوبَي كانبان وغانت، والميزانيات، والتكاليف، وسجل مخاطر لكل مبادرة |
| **TurboLens** | ذكاء هندسة المؤسسة المدعوم بالذكاء الاصطناعي — تحليل المزوّدين، واكتشاف التكرارات، وتقييم التحديث، ومعالج الذكاء الاصطناعي الهندسي ذي الخطوات الخمس، وعمليات فحص الامتثال مقابل EU AI Act / GDPR / NIS2 / DORA / SOC 2 / ISO 27001 |
| **تسليم هندسة المؤسسة** | سطح التسليم المتوافق مع TOGAF — بيانات عمل الهندسة، وسجلات قرارات الهندسة، وسجل المخاطر على مستوى المشهد |
| **SoAW** | Statement of Architecture Work — وثيقة TOGAF رسمية تحدّد نطاق مبادرة هندسية |
| **ADR** | Architecture Decision Record — يلتقط سياق القرار وبدائله وتبعاته، مع سير عمل الحالة وربط البطاقات |
| **سجل المخاطر** | سجل مخاطر TOGAF المرحلة G على مستوى المشهد، منفصل عن مخاطر PPM على مستوى المبادرة. يُنشئ إسناد المالك مهمة تلقائيًا |
| **بوابة إلكترونية** | عرض عام للقراءة فقط مبني على slug لجزء من مشهد هندسة المؤسسة — قابل للمشاركة دون تسجيل دخول |
| **خادم MCP** | وصول أدوات الذكاء الاصطناعي للقراءة فقط عبر Model Context Protocol — استعلم بيانات هندسة المؤسسة من Claude Desktop وCursor وGitHub Copilot وغيرها من عملاء MCP |
| **RBAC** | التحكّم في الوصول المستند إلى الأدوار — أدوار على مستوى التطبيق إضافةً إلى أدوار أصحاب المصلحة لكل بطاقة مع أكثر من 50 إذنًا دقيقًا |
